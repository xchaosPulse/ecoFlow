# بلوپرینت پروژه: داشبورد تعاملی اقتصاد شفاف ایران (Iran Open Economy Dashboard)

**وضعیت سند:** پیش‌نویس اولیه (v1.0)  
**نوع پروژه:** متن‌باز (Open Source) - تجسم داده (Data Visualization)  
**هدف:** شفاف‌سازی چرخه اقتصادی، نمایش بصری جریان پول، رانت، و اثرات تصمیم‌گیری‌های کلان بر معیشت مردم.

---

## ۱. نمای کلی پروژه (Executive Summary)
این پروژه یک پلتفرم تحت وب تعاملی است که اقتصاد ایران را نه به صورت جداول خشک، بلکه به صورت یک **ارگانیسم زنده** یا **ماشین مکانیکی** (شامل چرخ‌دنده‌ها، لوله‌های انتقال جریان پول و مخازن) شبیه‌سازی می‌کند.
هدف اصلی این است که کاربر بتواند با تغییر متغیرهای کلیدی (مانند حذف رانت ارزی، تغییر نرخ صادرات نفت، یا کاهش چاپ پول)، تأثیر مستقیم آن را بر متغیرهای پایین‌دستی (مانند یارانه نقدی، دستمزد کارگر، قدرت خرید و تورم) به صورت لحظه‌ای و انیمیشنی مشاهده کند.

### اهداف کلیدی:
*   **شفافیت بصری:** تبدیل اعداد انتزاعی بودجه و اقتصاد به گرافیک قابل فهم.
*   **تعامل‌پذیری (Interactivity):** امکان سناریوسازی توسط کاربر (What-if Analysis).
*   **استنادپذیری:** استفاده از داده‌های رسمی داخلی و بین‌المللی با ذکر منبع دقیق.
*   **معماری غیرمتمرکز:** میزبانی به صورت استاتیک روی گیت‌هاب برای جلوگیری از سانسور و کاهش هزینه نگهداری.

---

## ۲. معماری سیستم (System Architecture)

### ۲.۱. پشته تکنولوژی (Tech Stack)
*   **هسته (Core):** React.js (برای مدیریت کامپوننت‌ها و stateها).
*   **بصری‌سازی (Visualization):**
    *   `D3.js`: برای نمودارهای دقیق و جریان‌های داده.
    *   `Three.js` یا `React-Spring`: برای انیمیشن‌های تعاملی (چرخ‌دنده‌ها و جریان پول).
*   **مدیریت داده:** فایل‌های JSON/CSV استاتیک (بدون دیتابیس سمت سرور).
*   **هاستینگ:** GitHub Pages (استاتیک).
*   **زیرساخت به‌روزرسانی:** GitHub Actions (برای بیلد اتوماتیک پس از تغییر داده‌ها).

### ۲.۲. جریان داده (Data Flow Model)
مدل اقتصادی به صورت یک گراف جهت‌دار (DAG) طراحی می‌شود:
1.  **ورودی‌ها (Sources):** درآمد نفتی، مالیات، چاپ پول (پایه پولی).
2.  **توزیع (Distribution):** بودجه دولت، صندوق توسعه، رانت‌ها (ارزی/انرژی)، هزینه‌های جاری.
3.  **خروجی‌ها (Sinks):** یارانه‌ها، پروژه‌های عمرانی، حقوق و دستمزد، تورم (به عنوان نشت سیستم).

---

## ۳. استراتژی داده‌ها (Data Strategy)

داده‌ها به صورت "زنده" از APIهای دولتی خوانده نمی‌شوند (به دلیل ناپایداری). در عوض، از روش **Data Check-in** استفاده می‌شود.

### ۳.۱. منابع داده (Data Sources)
تیم نگهداری داده‌ها را از منابع زیر استخراج و در فایل‌های پروژه (JSON) به‌روزرسانی می‌کند:
*   **داخلی:**
    *   مرکز آمار ایران (نرخ تورم، GDP، اشتغال).
    *   بانک مرکزی (CBI): (نقدینگی، پایه پولی، ترازنامه).
    *   مرکز پژوهش‌های مجلس: (جزئیات لایحه بودجه سالانه).
    *   وزارت نفت / گمرک: (آمار صادرات).
*   **بین‌المللی (جهت اعتبارسنجی):**
    *   World Bank Open Data & IMF DataMapper (شاخص‌های کلان).
    *   OPEC / IEA (داده‌های انرژی و نفت).
    *   Transparency International (شاخص‌های تخمینی فساد/رانت).

### ۳.۲. ساختار ذخیره‌سازی داده (Schema)
داده‌ها در مسیر `/data/economics.json` ذخیره می‌شوند:

```json
{
  "last_updated": "2024-11-23",
  "fiscal_year": "1403",
  "parameters": {
    "oil_revenue": { "value": 55, "unit": "Billion USD", "source": "OPEC/MOP" },
    "exchange_rate_official": { "value": 28500, "unit": "Tomans", "source": "CBI" },
    "exchange_rate_market": { "value": 60000, "unit": "Tomans", "source": "Bonbast" },
    "money_printing_volume": { "value": 300, "unit": "Trillion Tomans", "source": "CBI" },
    "subsidy_energy_total": { "value": 80, "unit": "Billion USD", "source": "IEA" }
  }
}
```

### ۳.۳. فرآیند به‌روزرسانی (Contribution Workflow)
1.  یک **Issue** در گیت‌هاب باز می‌شود (مثلاً: انتشار گزارش جدید بانک مرکزی).
2.  تیم داده، اعداد جدید را استخراج می‌کند.
3.  یک **Pull Request** ارسال می‌شود که فایل `json` را تغییر می‌دهد.
4.  پس از Merge، سیستم CI/CD سایت را دوباره بیلد و منتشر می‌کند.

## ۴. طراحی رابط کاربری و تجربه کاربری (UI/UX)

### ۴.۱. داشبورد اصلی (Main View)
یک نمای گرافیکی سراسری که شبیه به نقشه یک پالایشگاه یا سیستم مکانیکی است:
*   **مخزن نفت:** نشان‌دهنده ورودی دلار نفتی.
*   **شیرهای فلکه (Valves):** نشان‌دهنده تخصیص بودجه (چقدر به دولت؟ چقدر به صندوق؟ چقدر به رانت؟).
*   **چرخ‌دنده‌های تورم:** هرچه سرعت چاپ پول بیشتر شود، این چرخ‌دنده‌ها قرمزتر و سریع‌تر می‌چرخند و ارزش پول خروجی (حقوق) را "می‌جوند".

### ۴.۲. پنل کنترل (Control Panel)
ابزارهای تعاملی برای کاربر:
*   **اسلایدر "رانت ارزی":** کاهش از ۳۰٪ به ۰٪ -> مشاهده افزایش بودجه عمرانی یا یارانه نقدی.
*   **اسلایدر "قیمت بنزین":** افزایش قیمت -> مشاهده کاهش قاچاق + افزایش تورم انتظاری + افزایش یارانه نقدی (نمایش ترید-آف‌ها).

### ۴.۳. بخش جزئیات (Drill-down)
با کلیک روی هر بخش (مثلاً بخش حقوق کارگر):
*   نمایش قدرت خرید واقعی در طول ۱۰ سال گذشته.
*   نمایش سهم هزینه‌های مسکن، خوراک و حمل‌ونقل در سبد خانوار.

## ۵. فازبندی توسعه (Development Roadmap)

### فاز ۱: MVP (حداقل محصول قابل ارائه)
*   پیاده‌سازی ساختار پروژه React.
*   ایجاد فایل JSON داده‌های پایه (بودجه ۱۴۰۳).
*   طراحی صفحه ساده با نمودارهای میله‌ای/دایره‌ای تعاملی (بدون انیمیشن پیچیده).
*   محاسبه‌گر ساده: "اگر رانت حذف شود، یارانه من چقدر می‌شود؟"

### فاز ۲: ویژوالیزیشن پیشرفته
*   اضافه کردن کتابخانه Three.js/D3.js.
*   پیاده‌سازی مدل انیمیشنی "چرخه پول".
*   اضافه کردن داده‌های تاریخی (سری زمانی ۱۰ ساله).

### فاز ۳: اتوماسیون و مشارکت
*   تکمیل مستندات برای مشارکت‌کنندگان (Data Contribution Guide).
*   نوشتن اسکریپت‌های پایتون (Scrapers) برای واکشی خودکار برخی داده‌های اوپن‌سورس جهانی.
*   نسخه انگلیسی برای مخاطبان بین‌المللی.

## ۶. نیازمندی‌های تیم (Team Roles)

برای اجرای این پروژه به نقش‌های زیر (حتی به صورت داوطلبانه) نیاز است:
1.  **Data Analyst/Economist:** جهت استخراج داده‌های صحیح از گزارش‌های پیچیده بودجه و بانک مرکزی و نگاشت روابط علت و معلولی.
2.  **Frontend Developer:** مسلط به React و کتابخانه‌های بصری‌سازی (D3/Three.js).
3.  **UI/UX Designer:** برای تبدیل مفاهیم پیچیده اقتصادی به المان‌های بصری ساده.
4.  **Maintainer:** جهت بررسی PRها و مدیریت مخزن گیت‌هاب.
